# Support

This is a small, community-built project maintained in spare time. There is no support contract
and no SLA — but issues do get read.

## Where to go

| You want to… | Go here |
| --- | --- |
| Report something broken | [Open a bug report](https://github.com/flaketrace/playwright-ai-triage/issues/new?template=bug_report.yml) |
| Ask for a feature | [Open a feature request](https://github.com/flaketrace/playwright-ai-triage/issues/new?template=feature_request.yml) |
| Ask a question, or show what you built | [Discussions](https://github.com/flaketrace/playwright-ai-triage/discussions) |
| Report a security vulnerability | **Not a public issue** — see [SECURITY.md](SECURITY.md) |

## Before you open an issue

- Check the [README](https://github.com/flaketrace/playwright-ai-triage#readme) — the full option
  surface and the "what data is sent where" breakdown are both there.
- Check the [roadmap](https://github.com/flaketrace/playwright-ai-triage/blob/main/docs/ROADMAP.md).
  If your request is already on it, say so in an issue anyway — demand is what reorders it.
- Reproduce on the latest published version if you can. The package is pre-1.0 and there are no
  maintenance branches.

## What helps a bug get fixed

The bug form asks for the reporter's stdout summary, your versions, and your reporter config.
Please fill it in rather than replacing it with prose — a misclassification report without the
failure's error text and the class the reporter produced usually can't be acted on.

**Check your paste for secrets before you post it.** Redaction is best-effort and pattern-based;
an unusual secret format may not be masked.

## Response expectations

Expect a few days. Security reports are prioritised over everything else.
